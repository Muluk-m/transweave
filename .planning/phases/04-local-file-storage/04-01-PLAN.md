Output: A working upload endpoint (POST /api/upload), a static file server (GET /uploads/*), and updated frontend code that uploads to and displays from the local server.
</objective>

<task type="auto">
  <name>Task 1: Create server-side file storage module with upload endpoint and static serving</name>
  <files>

  <action>
  1. Install @nestjs/serve-static:
     ```
     cd packages/server && pnpm add @nestjs/serve-static
     ```

  2. Create `packages/server/src/service/file-storage.service.ts`:
     - Injectable service implementing `OnModuleInit`
     - Constructor reads `process.env.UPLOAD_DIR` (default: `'./uploads'`) and resolves to absolute path via `path.resolve()`
     - `onModuleInit()` calls `mkdir(this.uploadDir, { recursive: true })` to ensure directory exists
     - `getUploadDir(): string` returns the resolved absolute path
     - `getFileUrl(filename: string): string` returns `/uploads/${filename}`
     - Use Logger to log the upload directory path on init

  3. Create `packages/server/src/controller/upload.controller.ts`:
     - Controller on route `'api/upload'`
     - Single `@Post()` endpoint protected with `@UseGuards(AuthGuard)`
     - Use `@UseInterceptors(FileInterceptor('file', { ... }))` with:
       - `storage: diskStorage({ destination: process.env.UPLOAD_DIR || './uploads', filename: (req, file, cb) => cb(null, uuidv4() + extname(file.originalname)) })`
       - `limits: { fileSize: 10 * 1024 * 1024 }` (10MB max)
       - `fileFilter` that accepts only image MIME types matching `/^image\//` -- reject others with BadRequestException('Only image files are allowed')
     - Handler receives `@UploadedFile() file: Express.Multer.File`
     - If `!file`, throw `BadRequestException('No file provided')`
     - Return `{ url: '/uploads/' + file.filename, name: file.originalname, size: file.size, type: file.mimetype }` -- matching the existing `UploadFile` interface on the frontend
     - Import `v4 as uuidv4` from `uuid` (already in dependencies), `extname` from `path`, `diskStorage` from `multer`, `FileInterceptor` from `@nestjs/platform-express`

  4. Create `packages/server/src/modules/file-storage.module.ts`:
     - Import `ServeStaticModule.forRoot()` with:
       - `rootPath: resolve(process.env.UPLOAD_DIR || './uploads')` -- use `path.resolve()`
       - `serveRoot: '/uploads'`
       - `serveStaticOptions: { index: false, fallthrough: false }`
     - Register `UploadController` in controllers array
     - Register `FileStorageService` in providers array
     - Export `FileStorageService`

  5. Update `packages/server/src/app.module.ts`:
     - Add `import { FileStorageModule } from './modules/file-storage.module'`
     - Add `FileStorageModule` to the `imports` array (alongside PassportModule, DatabaseModule, etc.)
     - Do NOT add UploadController or FileStorageService to the AppModule directly -- they come through FileStorageModule

  IMPORTANT NOTES:
  - The `diskStorage` destination in the FileInterceptor decorator is resolved at module load time. Use `process.env.UPLOAD_DIR || './uploads'` directly (not via service injection). The service is for runtime directory management and for other code that needs the path.
  - The `resolve()` in ServeStaticModule.forRoot must use `path.resolve()` to convert relative paths to absolute -- serve-static requires absolute paths.
  - Do NOT add CORS configuration or body parser size limits -- Multer handles its own parsing separately from the NestJS body parser.
  - Keep the `@types/multer` types that come transitively through `@nestjs/platform-express` -- do NOT install `@types/multer` separately as it can cause version conflicts.

<task type="auto">
  <name>Task 2: Update frontend to upload files to local server and display from local URLs</name>
  <files>packages/web/api/upload.ts</files>
  <action>
  Rewrite `packages/web/api/upload.ts` to target the local NestJS server instead of the external CDN.

  1. Keep the existing `UploadFile` and `UploadResponse` interfaces unchanged (the server returns matching shape).

  2. Rewrite `uploadImage(file: File): Promise<UploadFile>`:
     - Create FormData, append file as `'file'` (not `'files'` -- must match server's FileInterceptor('file'))
     - POST to `${process.env.NEXT_PUBLIC_API_URL || 'http://localhost:3001'}/api/upload`
     - Include `Authorization: Bearer ${localStorage.getItem('authToken')}` header
     - Do NOT include Content-Type header (browser sets multipart boundary automatically)
     - Parse response as JSON and return directly (server returns { url, name, size, type } which matches UploadFile)
     - On non-ok response, throw `new Error('Failed to upload file')`

  3. Rewrite `uploadFiles(files: File[]): Promise<UploadFile[]>`:
     - Upload each file sequentially by calling `uploadImage()` in a for loop
     - Collect and return results array
     - Sequential upload is simpler and avoids overwhelming the server

  4. Rewrite `getImageUrl(url: string): string`:
     - If url starts with 'http://' or 'https://', return as-is (backward compatibility with any legacy absolute URLs)
     - Otherwise, prepend `process.env.NEXT_PUBLIC_API_URL || 'http://localhost:3001'` to the relative path
     - The relative path will be like `/uploads/abc123.png` (from server response)
     - Result: `http://localhost:3001/uploads/abc123.png`

  5. Remove the `UploadResponse` interface if it was only used for the old CDN response parsing (the new flow doesn't need it). Keep it if it's imported elsewhere.

  6. Remove all references to `https://qlj-devhub-homepage.qiliangjia.one` -- this is the external CDN being replaced.

  7. Remove all code that parses `f.urls.r2` or `Object.values(f.urls)` -- the new server returns a flat response, not the nested CDN format.

  IMPORTANT NOTES:
  - The FormData field name MUST be `'file'` (singular) to match `FileInterceptor('file')` on the server. The old code used `'files'` (plural) -- this must change.
  - Do NOT use the `apiClient` from `lib/api.ts` for file uploads because it may set Content-Type headers that interfere with multipart boundary detection. Use raw `fetch()` directly.
  - The existing consumers (TokenFormDrawer.tsx, TokenTable.tsx) import `uploadImage` and `getImageUrl` -- the function signatures must remain the same so consumers don't need changes.
  </action>
