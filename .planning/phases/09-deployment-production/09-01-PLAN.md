2. Create `.env.example` at project root documenting ALL environment variables. Group by category with comments. Include these variables:
   - `DATABASE_URL` - PostgreSQL connection string (required for Docker; omit for PGlite)
   - `POSTGRES_DB` - Database name (default: i18n)
   - `POSTGRES_USER` - Database user (default: i18n)
   - `POSTGRES_PASSWORD` - Database password (required, no default)
   - `JWT_SECRET` - JWT signing key (required, include generation command: `openssl rand -base64 64`)
   - `PORT` - Server port (default: 3001)
   - `UPLOAD_DIR` - Upload directory path (default: ./uploads)
   - `NEXT_PUBLIC_API_URL` - Backend API URL for browser (default: http://localhost:3001, note: BUILD-TIME variable)
   - `NEXT_INTERNAL_API_URL` - Backend API URL for SSR inside Docker network (default: http://server:3001)
   - `WEB_PORT` - Host port for web UI (default: 3000)
   - `AI_PROVIDER` - AI translation provider (optional, empty = disabled)
   - `AI_API_KEY` - AI provider API key (optional)
   - `PGLITE_DATA_DIR` - PGlite data directory for non-Docker mode (default: ./data/pglite)

3. Create `packages/server/src/health/health.controller.ts`:
   ```typescript
   import { Controller, Get } from '@nestjs/common';

   @Controller('health')
   export class HealthController {
     @Get()
     check() {
       return { status: 'ok', timestamp: new Date().toISOString() };
     }
   }
   ```

4. Create `packages/server/src/health/health.module.ts`:
   ```typescript
   import { Module } from '@nestjs/common';
   import { HealthController } from './health.controller';

   @Module({
     controllers: [HealthController],
   })
   export class HealthModule {}
   ```

5. Modify `packages/server/src/app.module.ts` to import `HealthModule`. Add it to the `imports` array. The health module has no dependencies so it can be added unconditionally.

<task type="auto">
  <name>Task 2: Rewrite Dockerfiles and docker-compose.yml for PostgreSQL</name>
  <files>packages/server/Dockerfile, packages/web/Dockerfile, docker-compose.yml</files>
  <action>
1. Rewrite `packages/server/Dockerfile` (move from project root `Dockerfile.server` to `packages/server/Dockerfile`). Use four-stage multi-stage build:

   **Stage 1 (base):** `node:22-alpine`, enable corepack, prepare pnpm@10.8.0 (pinned to match packageManager in root package.json), set WORKDIR /app.

   **Stage 2 (deps):** FROM base. COPY package.json, pnpm-workspace.yaml, pnpm-lock.yaml from root. COPY packages/server/package.json. Run `pnpm install --frozen-lockfile`.

   **Stage 3 (builder):** FROM deps. COPY packages/server source. Run `pnpm --filter qlj-i18n-server build`.

   **Stage 4 (runner):** FROM node:22-alpine. Enable corepack, prepare pnpm@10.8.0. COPY package.json, pnpm-workspace.yaml, pnpm-lock.yaml, packages/server/package.json. Run `pnpm install --frozen-lockfile --prod`. COPY dist from builder. Set NODE_ENV=production, PORT=3001. EXPOSE 3001. Add HEALTHCHECK using wget to http://localhost:3001/health. Use exec-form CMD: `CMD ["node", "packages/server/dist/main.js"]`.

   IMPORTANT: Do NOT use `pnpm@latest`. Pin to `pnpm@10.8.0`. Do NOT copy node_modules from builder AND install in runner (choose one -- install fresh with --prod in runner). Use exec form CMD, not shell form.

2. Rewrite `packages/web/Dockerfile` (move from project root `Dockerfile.web` to `packages/web/Dockerfile`). Similar four-stage build:

   **Stage 1 (base):** Same as server.

   **Stage 2 (deps):** Same pattern, but COPY packages/web/package.json.

   **Stage 3 (builder):** FROM deps. COPY packages/web source. Accept build arg: `ARG NEXT_PUBLIC_API_URL=http://localhost:3001`. Set as ENV. Run `pnpm --filter web build`. Next.js standalone output should be used if next.config.mjs has `output: 'standalone'`. If not, use standard build.

   **Stage 4 (runner):** FROM node:22-alpine. COPY .next and public from builder. COPY node_modules for production. Set NODE_ENV=production, PORT=3000. EXPOSE 3000. Use exec-form CMD: `CMD ["node", "packages/web/node_modules/.bin/next", "start", "packages/web"]` or equivalent pnpm start command.

   IMPORTANT: `NEXT_PUBLIC_API_URL` MUST be a build ARG, not just an environment variable, because Next.js bakes it at build time.

3. Rewrite `docker-compose.yml` completely. Remove all MongoDB/bondma references. Create three services:

   **postgres service:**
   - Image: `postgres:17-alpine`
   - Environment: POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD from .env with defaults (DB=i18n, USER=i18n, PASSWORD required via `${POSTGRES_PASSWORD:?...}`)
   - Volume: `pgdata:/var/lib/postgresql/data`
   - Health check: `pg_isready -U ${POSTGRES_USER:-i18n} -d ${POSTGRES_DB:-i18n}`, interval 5s, timeout 5s, retries 5
   - Restart: unless-stopped
   - No exposed ports (only accessible within Docker network)

   **server service:**
   - Build: context `.`, dockerfile `packages/server/Dockerfile`
   - depends_on postgres with `condition: service_healthy`
   - Environment: DATABASE_URL constructed from POSTGRES_* vars (`postgresql://${POSTGRES_USER:-i18n}:${POSTGRES_PASSWORD}@postgres:5432/${POSTGRES_DB:-i18n}`), JWT_SECRET (required), PORT=3001, UPLOAD_DIR=/app/uploads
   - Volume: `uploads:/app/uploads`
   - Restart: unless-stopped
   - No exposed ports (accessed by web service or through web reverse proxy)

   **web service:**
   - Build: context `.`, dockerfile `packages/web/Dockerfile`, args: NEXT_PUBLIC_API_URL from env
   - depends_on: server
   - Environment: NEXT_INTERNAL_API_URL=http://server:3001
   - Ports: `${WEB_PORT:-3000}:3000`
   - Restart: unless-stopped

   **Volumes:**
   - `pgdata:` (PostgreSQL data)
   - `uploads:` (uploaded files)

   Do NOT include a networks section (Docker Compose v2 creates a default network automatically). Do NOT include any MongoDB references, bondma references, or commented-out services.

4. Delete the old root-level `Dockerfile.server` and `Dockerfile.web` files since the new ones are in packages/server/ and packages/web/ respectively.
  </action>
