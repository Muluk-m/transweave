Add the following methods to `TokenService` (created in Plan 01):

**1. `search(projectId: string, options: TokenSearchOptions): Promise<{ tokens: Token[], total: number }>`**

`TokenSearchOptions` type:
```typescript
interface TokenSearchOptions {
  query?: string;          // Full-text search across key and translation values
  module?: string;         // Filter by module code ('__no_module__' for tokens without module)
  status?: 'all' | 'completed' | 'incomplete'; // Language completion status
  language?: string;       // Specific language for status filter (e.g., 'fr' means "filter by French completion")
  tags?: string[];         // Filter by tags (tokens must have ALL specified tags)
  page?: number;           // Default: 1
  perPage?: number;        // Default: 50, max: 200
  sortBy?: string;         // 'key' | 'createdAt' | 'updatedAt', default: 'createdAt'
  sortOrder?: 'asc' | 'desc'; // Default: 'desc'
}
```

**SQL query construction:**

Build a dynamic WHERE clause using Drizzle's `and()` and `or()`:

a) **Base condition:** `WHERE project_id = $projectId`

b) **Text search (query param):** Add condition:
```sql
AND (
  key ILIKE '%query%'
  OR EXISTS (
    SELECT 1 FROM jsonb_each_text(translations) AS t(lang, val)
    WHERE t.val ILIKE '%query%'
  )
)
```
Use Drizzle's `sql` template with parameterized query string to prevent injection. The `jsonb_each_text()` function extracts all key-value pairs from the JSONB translations column and checks each value.

c) **Module filter (module param):**
- If `module === '__no_module__'`: `AND (module = '' OR module IS NULL)`
- Otherwise: `AND module = $module`

d) **Completion status filter (status + language params):**
- Fetch the project's languages list first (needed to determine what "complete" means).
- If status = 'completed' AND no specific language:
  `AND` for each lang in project.languages: `translations->>$lang IS NOT NULL AND translations->>$lang != ''`
- If status = 'completed' AND specific language provided:
  `AND translations->>$language IS NOT NULL AND translations->>$language != ''`
- If status = 'incomplete' AND no specific language:
  `AND` EXISTS at least one lang where `translations->>$lang IS NULL OR translations->>$lang = ''`
- If status = 'incomplete' AND specific language provided:
  `AND (translations->>$language IS NULL OR translations->>$language = '')`

e) **Tags filter (tags param):**
  `AND tags @> $tags::jsonb` (JSONB contains operator -- token's tags must contain all specified tags)

f) **Sorting:** Apply ORDER BY based on sortBy and sortOrder.

g) **Pagination:** Apply LIMIT and OFFSET. Also run a COUNT(*) query with the same WHERE conditions (without LIMIT/OFFSET) to return total count.

Return `{ tokens: Token[], total: number }` where tokens have their history populated (query token_history joined with users).

**2. `getLanguageCompletion(projectId: string): Promise<LanguageProgress[]>`**

`LanguageProgress` type:
```typescript
interface LanguageProgress {
  language: string;      // Language code (e.g., 'en', 'zh', 'fr')
  total: number;         // Total number of tokens in the project
  completed: number;     // Number of tokens with non-empty value for this language
  percentage: number;    // Math.round((completed / total) * 100)
}
```

Implementation:
1. Fetch project to get its languages array.
2. Count total tokens for the project: `SELECT COUNT(*) FROM tokens WHERE project_id = $projectId`
3. For each language in the project's languages array, count completed tokens:
   ```sql
   SELECT COUNT(*) FROM tokens
   WHERE project_id = $projectId
   AND translations->>$lang IS NOT NULL
   AND translations->>$lang != ''
   ```
4. Compute percentage for each language.
5. Return array of LanguageProgress objects.

**Optimization note:** If the project has many languages (e.g., 20+), this could be done in a single query using lateral joins or CASE expressions to avoid N separate queries. But for v1, the simple loop is clear and correct. Optimize later if profiling shows a bottleneck (unlikely for <50 languages).

**Important:** These methods use the repository layer, not raw Drizzle queries directly. If the repository from Phase 2 does not have these query capabilities, extend the repository with the necessary methods (parameterized query builder).

Add the following endpoints to `TokenController` (created in Plan 01):

**1. `GET /api/tokens/:projectId/search` -- Search and filter tokens**

Query parameters:
- `q` (string, optional): Search query text
- `module` (string, optional): Module code filter (use '__no_module__' for unassigned)
- `status` ('all' | 'completed' | 'incomplete', optional, default: 'all'): Completion status filter
- `language` (string, optional): Specific language code for status filter
- `tags` (string, optional): Comma-separated tag names
- `page` (number, optional, default: 1): Page number
- `perPage` (number, optional, default: 50): Items per page
- `sortBy` (string, optional, default: 'createdAt'): Sort field
- `sortOrder` ('asc' | 'desc', optional, default: 'desc'): Sort direction

Implementation:
```typescript
@Get(':projectId/search')
@UseGuards(AuthGuard)
async searchTokens(
  @Param('projectId') projectId: string,
  @Query('q') query?: string,
  @Query('module') module?: string,
  @Query('status') status?: 'all' | 'completed' | 'incomplete',
  @Query('language') language?: string,
  @Query('tags') tags?: string,
  @Query('page') page?: string,
  @Query('perPage') perPage?: string,
  @Query('sortBy') sortBy?: string,
  @Query('sortOrder') sortOrder?: 'asc' | 'desc',
  @CurrentUser() user: UserPayload,
) {
  // Verify permission
  await this.checkPermission(projectId, user.userId);

  return this.tokenService.search(projectId, {
    query,
    module,
    status: status || 'all',
    language,
    tags: tags ? tags.split(',').map(t => t.trim()) : undefined,
    page: page ? parseInt(page, 10) : 1,
    perPage: perPage ? Math.min(parseInt(perPage, 10), 200) : 50,
    sortBy: sortBy || 'createdAt',
    sortOrder: sortOrder || 'desc',
  });
}
```

Response shape:
```json
{
  "tokens": [...],
  "total": 150,
  "page": 1,
  "perPage": 50,
  "totalPages": 3
}
```

private async checkPermission(projectId: string, userId: string) {
