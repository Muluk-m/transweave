  <action>
    1. **Create ApiKey schema** (`packages/server/src/models/schemas/api-key.schema.ts`):
       - Use the same Mongoose schema pattern as other models (import `baseSchemaOptions`, `addVirtualId` from `../base.schema`)
       - Fields:
         - `_id`: ObjectId (auto)
         - `name`: string, required (user-given label, e.g. "CI Pipeline")
         - `keyPrefix`: string, required (first 13 chars of key: "qlji_" + 8 hex chars, for display identification)
         - `keyHash`: string, required (bcrypt hash of full key)
         - `userId`: ObjectId ref to User, required
         - `scopes`: string array, default `['read', 'write']`
         - `expiresAt`: Date, optional (null = never expires)
         - `lastUsedAt`: Date, optional
         - `createdAt`: Date
         - `updatedAt`: Date
         - Virtual `id` field
       - Export `ApiKey`, `ApiKeyDocument`, `ApiKeySchema`

    2. **Update models barrel** (`packages/server/src/models/index.ts`):
       - Add `export * from './schemas/api-key.schema';`

    3. **Register schema in DatabaseModule** (`packages/server/src/modules/database.module.ts`):
       - Add `{ name: ApiKey.name, schema: ApiKeySchema }` to the MongooseModule.forFeature array

    4. **Create ApiKeyService** (`packages/server/src/service/api-key.service.ts`):
       - Inject `@InjectModel(ApiKey.name)` for Mongoose model
       - `createKey(userId: string, name: string, scopes?: string[], expiresAt?: Date)`:
         - Generate key: `qlji_` + `crypto.randomBytes(32).toString('hex')` (total ~69 chars)
         - Compute prefix: first 13 chars of key
         - Hash key with `hashPassword()` from `src/utils/crypto`
         - Save to DB: `{ name, keyPrefix, keyHash, userId, scopes, expiresAt }`
         - Return `{ key: <full key shown once>, id, name, keyPrefix, scopes, expiresAt, createdAt }` -- full key is ONLY returned at creation
       - `validateKey(providedKey: string)`:
         - Extract prefix from providedKey (first 13 chars)
         - Find all non-expired keys matching that prefix: `{ keyPrefix, $or: [{ expiresAt: null }, { expiresAt: { $gt: new Date() } }] }`
         - For each candidate, call `verifyPassword(providedKey, candidate.keyHash)`
         - If match found, update `lastUsedAt` to now and return `{ userId, userName, userEmail, scopes }` (populate user)
         - If no match, return null
       - `listKeys(userId: string)`:
         - Find all keys for user, select `id, name, keyPrefix, scopes, expiresAt, lastUsedAt, createdAt` (never return keyHash)
         - Sort by createdAt descending
       - `deleteKey(keyId: string, userId: string)`:
         - Find and delete key where `{ _id: keyId, userId }` (ownership check)
         - Return success/failure

    5. **Modify AuthGuard to accept API keys** (`packages/server/src/jwt/guard.ts`):
       - Replace the simple Passport guard with a custom `CanActivate` guard
       - Import `JwtService` from `@nestjs/jwt` and `ApiKeyService`
       - Logic in `canActivate(context: ExecutionContext)`:
         a. Extract `Authorization` header from request
         b. If missing, return false
         c. Parse `Bearer <token>` format
         d. If token starts with `qlji_`:
            - Call `apiKeyService.validateKey(token)`
            - If valid, set `request.user = { userId, email, name, avatar: '', authType: 'api_key' }`
            - Return true/false
         e. Otherwise, try JWT validation via `jwtService.verify(token)` with secret from env
            - If valid, set `request.user = { userId: payload.userId, email: payload.email, name: payload.name, avatar: payload.avatar, authType: 'jwt' }`
            - Return true/false
       - Keep the class name as `AuthGuard` so existing `@UseGuards(AuthGuard)` decorators still work
       - Since `AuthGuard` can no longer extend PassportStrategy's guard, update `JwtStrategy` to be unused (but keep it for now as a reference; it can be removed in cleanup)
       - The guard needs to be `@Injectable()` and receive dependencies via constructor injection. Because guards used with `@UseGuards()` are instantiated by NestJS DI, this works if the guard is provided in AppModule.

    6. **Update AppModule** (`packages/server/src/app.module.ts`):
       - Add `ApiKeyService` to providers array
       - Add `ApiKeyController` to controllers array (created in Task 2, but register here now)
       - Import `ApiKey` and `ApiKeySchema` from models

       - Route prefix: `api/api-keys`
       - All endpoints protected with `@UseGuards(AuthGuard)` (JWT auth for web UI access)
       - `POST /` - Create API key:
         - Body: `{ name: string, scopes?: string[], expiresAt?: string }`
         - Calls `apiKeyService.createKey(user.userId, name, scopes, expiresAt)`
         - Returns `{ success: true, apiKey: { key, id, name, keyPrefix, scopes, expiresAt, createdAt } }`
         - NOTE: `key` field contains the full API key -- this is the ONLY time it's returned
       - `GET /` - List API keys:
         - Calls `apiKeyService.listKeys(user.userId)`
         - Returns array of keys (without keyHash or full key)
       - `DELETE /:id` - Revoke API key:
         - Calls `apiKeyService.deleteKey(id, user.userId)`
         - Returns `{ success: true }`

    2. **Create frontend API client** (`packages/web/api/api-key.ts`):

       - `listApiKeys()` -> GET `/api/api-keys`
       - `deleteApiKey(id: string)` -> DELETE `/api/api-keys/${id}`
       - Use `apiClient` from `@/lib/api`
