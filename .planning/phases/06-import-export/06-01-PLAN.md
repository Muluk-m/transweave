Output: Format handler modules for XLIFF and Gettext with full round-trip support (parse and serialize), integrated into the existing `importFrom.ts` and `exportTo.ts` utility files. All existing format support continues to work unchanged.
</objective>

  <behavior>
    XLIFF Parser:
    - Parses valid XLIFF 1.2 file with source and target into { lang: { key: value } }
    - Uses trans-unit `id` attribute as key (falls back to `resname` if available)
    - Extracts source-language and target-language from file attributes
    - Returns empty result for empty XLIFF body
    - Throws descriptive error for malformed XLIFF

    XLIFF Serializer:
    - Serializes tokens array into valid XLIFF 1.2 XML string
    - Includes source-language and target-language attributes on file element
    - Each token becomes a trans-unit with source and target elements
    - Handles special XML characters (ampersand, angle brackets, quotes) via escaping
    - Produces valid XML that can be re-parsed by the parser

    Gettext Parser:
    - Parses valid .po file with msgid/msgstr pairs into { lang: { key: value } }
    - Extracts language from PO header "Language:" field
    - Skips the empty-msgid header entry
    - Handles msgctxt (context) by prefixing key as "context.msgid"
    - Handles multiline msgstr (concatenated quoted strings)
    - Ignores plural forms (msgid_plural / msgstr[N]) with warning -- imports msgstr[0] only

    Gettext Serializer:
    - Serializes tokens into valid .po file string
    - Includes proper PO header with Content-Type, Language, MIME-Version
    - Each token becomes a msgid/msgstr pair
    - Handles special characters in values (newlines, quotes)

    Round-trip:
    - export(parse(xliff_content)) produces equivalent translations
    - export(parse(po_content)) produces equivalent translations
  </behavior>
