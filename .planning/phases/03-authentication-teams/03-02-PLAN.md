<task type="auto">
  <name>Task 1: Migrate team.service.ts to repositories</name>
  <files>packages/server/src/service/team.service.ts</files>
  <action>
Full rewrite of team.service.ts to use repository layer instead of Mongoose models.

**Remove:**
- `@InjectModel(Team.name) private teamModel: Model<TeamDocument>`
- `@InjectModel(User.name) private userModel: Model<UserDocument>`
- `@InjectModel(Membership.name) private membershipModel: Model<MembershipDocument>`
- `private mongooseService: MongooseService`
- All Mongoose-specific imports (`Model`, `Types`, `InjectModel`, etc.)
- All `.populate()` chains
- All `session.withTransaction()` patterns
- All `$addToSet` operations
- All `.toObject()` calls

**Inject instead:**
- `TeamRepository`
- `MembershipRepository`
- `@Inject(DRIZZLE) private db: DrizzleInstance` (for transactions)

**Rewrite methods:**

`createTeam(data: { name: string; url: string; userId: string })`:
- Use a Drizzle transaction:
  ```typescript
  return this.db.transaction(async (tx) => {
    // 1. Create team
    const team = await this.teamRepo.create({ name, url }, tx);
    // 2. Create owner membership
    await this.membershipRepo.create({
      userId: data.userId,
      teamId: team.id,
      role: 'owner',
    }, tx);
    // 3. Return team with members
    return this.teamRepo.findWithMembers(team.id, tx);
  });
  ```
  Note: Repository methods should accept an optional transaction parameter. If Phase 2 repositories don't support this, pass the `tx` object directly for queries within the transaction scope. The approach depends on how Phase 2 implemented transactions. Check the repository base class pattern.

  If repositories don't accept transactions, use the Drizzle db instance directly within the transaction:
  ```typescript
  return this.db.transaction(async (tx) => {
    const [team] = await tx.insert(teams).values({ name, url }).returning();
    await tx.insert(memberships).values({ userId: data.userId, teamId: team.id, role: 'owner' });
    // ... query and return
  });
  ```

`findAllTeams()`:
- Return all teams with their memberships and user info.
- Use repository: `this.teamRepo.findAllWithMembers()` or construct a query that joins teams -> memberships -> users.
- The old code used nested `.populate()`. Replace with a JOIN or two queries:
  1. Fetch teams
  2. For each team, fetch memberships with user data
- Or use Drizzle's relational query API if configured in Phase 2.

`findTeamsByUserId(userId: string)`:
- Find memberships for user, then fetch those teams with all their memberships populated.
- Old pattern: find membership IDs, then find teams containing those IDs.
- New pattern: `this.membershipRepo.findByUserId(userId)` -> extract teamIds -> `this.teamRepo.findByIds(teamIds)` with members populated. Or use a single JOIN query.

`findTeamById(id: string)`:
- `this.teamRepo.findWithMembers(id)` -- returns team with memberships and user info.

`updateTeam(id: string, data: { name?: string; url?: string })`:
- `this.teamRepo.update(id, data)`.

`deleteTeam(id: string)`:
- Use transaction:
  1. Delete all memberships for team: `this.membershipRepo.deleteByTeamId(id)`
  2. Delete team: `this.teamRepo.delete(id)`

`getTeamMembers(teamId: string)`:
- Fetch memberships with user data for team.
- Map to `{ ...user, role: membership.role }` format (matching current response shape).
- Use `this.membershipRepo.findByTeamId(teamId)` which should return memberships with populated user fields.

**Key difference from Mongoose:** No more `memberships` array on the Team document. Memberships are queried from the memberships table using teamId foreign key. The response shape should remain the same for frontend compatibility.

<task type="auto">
  <name>Task 2: Migrate membership.service.ts to repositories</name>
  <files>packages/server/src/service/membership.service.ts</files>
  <action>
Full rewrite of membership.service.ts to use repository layer instead of Mongoose models.

**Remove:**
- `@InjectModel(Membership.name) private membershipModel: Model<MembershipDocument>`
- `@InjectModel(User.name) private userModel: Model<UserDocument>`
- `@InjectModel(Team.name) private teamModel: Model<TeamDocument>`
- `private mongooseService: MongooseService`
- All Mongoose-specific imports
- All `.populate()` chains
- All `session`/transaction from withTransaction
- All `$addToSet` operations
- Import of `withTransaction` from `src/utils/transaction`

**Inject instead:**
- `MembershipRepository`

**Rewrite methods:**

`createMembership(data: { userId: string; teamId: string; role: string })`:
- Use `this.membershipRepo.upsert(data)` -- upsert handles the "create or update if exists" pattern.
- The old code used `findOneAndUpdate` with `upsert: true`, then also pushed the membership ID into team.memberships array and user.memberships array. In the new relational model, there are NO membership arrays on teams or users -- memberships are queried via foreign keys. So this method only needs to upsert the membership row.
- Return the membership with user data populated (for frontend).
- Old code: returned membership populated with user. New code: `this.membershipRepo.findByTeamAndUser(data.teamId, data.userId)` with user JOIN after upsert.

`isMember(teamId: string, userId: string): Promise<boolean>`:
- `const m = await this.membershipRepo.findByTeamAndUser(teamId, userId); return !!m;`

`isOwner(teamId: string, userId: string): Promise<boolean>`:
- `const m = await this.membershipRepo.findByTeamAndUser(teamId, userId); return m?.role === 'owner';`

`isManagerOrOwner(teamId: string, userId: string): Promise<boolean>`:
- `const m = await this.membershipRepo.findByTeamAndUser(teamId, userId); return m?.role === 'owner' || m?.role === 'manager';`

`getUserRole(teamId: string, userId: string): Promise<string | null>`:
- `const m = await this.membershipRepo.findByTeamAndUser(teamId, userId); return m?.role ?? null;`

`getTeamMembers(teamId: string)`:
- `this.membershipRepo.findByTeamId(teamId)` -- should return memberships with user data (name, email, avatar).

`getUserMemberships(userId: string)`:
- `this.membershipRepo.findByUserId(userId)` -- should return memberships with team data.

`updateMemberRole(teamId: string, memberId: string, newRole: string)`:
- `this.membershipRepo.updateRole(teamId, memberId, newRole)`.
- Note: The `memberId` parameter in current code is a membership document _id. In the new schema this is the membership row's UUID id. Verify the frontend sends the correct ID type.

`removeMember(teamId: string, memberId: string)`:
- `this.membershipRepo.delete(teamId, memberId)`.
- Return type can be `Promise<void>` instead of `Promise<DeleteResult>` (remove Mongoose DeleteResult type).

`findMembership(teamId: string, userId: string)`:
- `this.membershipRepo.findByTeamAndUser(teamId, userId)`.

**Critical note:** The old code pushed membership IDs into `team.memberships[]` and `user.memberships[]` arrays. In the relational model, these arrays DO NOT EXIST on the team/user tables. Memberships are queried via the memberships table using foreign keys. This is a fundamental change from the document model. Do NOT try to maintain these arrays.
