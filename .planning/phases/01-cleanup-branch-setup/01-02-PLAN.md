<task type="auto">
  <name>Task 2: Create clean orphan branch and verify with secret scanner</name>
  <files>
    (git operations -- no file modifications, creates new branch)
  </files>
  <action>
    Create an orphan git branch that contains all current source files but ZERO commit history. This ensures no secrets from past commits leak to the open-source repository.

    **Step 1: Install gitleaks (if not available)**
    ```bash
    # Check if gitleaks is installed
    which gitleaks || brew install gitleaks
    ```

    **Step 2: Pre-scan current files on master**
    Before creating the orphan branch, run gitleaks on the current working tree (not history) to verify Plan 01 cleanup was successful:
    ```bash
    gitleaks detect --source . --no-git --verbose
    ```
    If this returns findings, STOP and fix them before proceeding. Do NOT create the orphan branch with secrets in source files.

    **Step 3: Create the orphan branch**
    ```bash
    # Save current branch name
    CURRENT_BRANCH=$(git branch --show-current)

    # Create orphan branch (no parent commits)
    git checkout --orphan opensource

    # The orphan branch has all files staged. We need to commit them.
    # First, make sure .gitignore is applied by unstaging everything, then re-adding
    git rm -r --cached . 2>/dev/null
    git add .

    # Verify .env is NOT staged
    git status | grep -i '.env' && echo "WARNING: .env files are staged!" || echo "OK: no .env files staged"

    # Create the initial commit
    git commit -m "Initial commit: clean codebase for open-source release

    Proprietary code removed:
    - Feishu OAuth integration
    - Internal company references (qiliangjia, bondma)
    - Hardcoded API keys and secrets
    - Internal CDN URLs

    Clean orphan branch with no inherited history."
    ```

    **Step 4: Verify the orphan branch**
    ```bash
    # Verify no parent commits (should show exactly 1 commit)
    git log --oneline | wc -l
    # Expected: 1

    # Verify no .env files committed
    git ls-files | grep -i '.env' | grep -v '.env.example' | grep -v '.gitignore'
    # Expected: no output

    # Run full gitleaks scan on the branch (including the one commit)
    gitleaks detect --source . --verbose
    # Expected: zero findings
    ```

    **Step 5: Switch back to master**
    ```bash
    git checkout $CURRENT_BRANCH
    ```

    **Important notes:**
    - The orphan branch has NO connection to master's history. `git log` on `opensource` shows only 1 commit.
    - The `.planning/` directory should NOT be on the opensource branch (excluded by .gitignore update in Task 1).
    - If gitleaks finds issues, fix them on master first, then recreate the orphan branch.
    - The `init/` directory (MongoDB replica set init) will be present but will be removed in Phase 2 when MongoDB is replaced.
  </action>
