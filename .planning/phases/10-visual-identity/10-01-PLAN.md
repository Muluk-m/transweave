  <action>
Design and write two SVG files implementing the Transweave weaving icon.

**Visual concept:** Interlocked rings motif — three overlapping circles arranged diagonally, each ring partially "going under" the next, suggesting thread interlocking and multi-language weaving. This reads at 16px (only the teal/indigo silhouette registers) and scales cleanly to 512px.

**Icon geometry (32x32 viewBox):**
- Rounded square background: `<rect width="32" height="32" rx="6" fill="url(#tw-icon-gradient)"/>`
- Three interlocked rings in white, arranged at 45°: three circles of radius ~5, centers at (10,10), (16,16), (22,22), each circle rendered as a stroke (stroke-width="2.5", fill="none", stroke="white") with opacity creating the over-under effect — use stroke-opacity="1.0" for the foreground arc and "0.35" for the background arc portion
- Alternatively: implement as two overlapping diagonal bands (rounded rectangles at 135° rotation) crossing in the center, white fill, creating a literal "weave" X pattern with one band appearing to go over the other via z-order

**Minimum readability rules:**
- No more than 4 distinct SVG path/shape elements
- All strokes >= 2px at 32x32 (translates to >=1px at 16px)
- Test mentally: at 16x16 only the background color and dominant shape silhouette remain — ensure the silhouette is recognizable

**`packages/web/public/logo.svg`** — icon-only, no dark mode CSS:
```xml
<svg viewBox="0 0 32 32" xmlns="http://www.w3.org/2000/svg" fill="none">
  <rect width="32" height="32" rx="6" fill="url(#tw-icon-gradient)"/>
  <!-- weaving motif paths, white on gradient background -->
  <defs>
    <linearGradient id="tw-icon-gradient" x1="0" y1="0" x2="32" y2="32" gradientUnits="userSpaceOnUse">
      <stop stop-color="#14b8a6"/>
      <stop offset="1" stop-color="#6366f1"/>
    </linearGradient>
  </defs>
</svg>
```

**`packages/web/app/icon.svg`** — same icon paths but WITH dark mode CSS media query:
```xml
<svg viewBox="0 0 32 32" xmlns="http://www.w3.org/2000/svg" fill="none">
  <style>
    .bg { fill: #0d9488; }
    .fg { fill: none; stroke: white; }
    @media (prefers-color-scheme: dark) {
      .bg { fill: #14b8a6; }
    }
  </style>
  <rect width="32" height="32" rx="6" class="bg"/>
  <!-- weaving motif paths using class="fg" -->
</svg>
```

Note: `app/icon.svg` must NOT use `fill="url(#tw-icon-gradient)"` on the rect — gradients via `<defs>` work when inlined in HTML but the `app/icon.svg` is loaded as an external SVG via `<link>` (file convention), so gradients in `<defs>` DO work for self-contained SVGs. The solid teal background with class-based dark mode switching is simpler and more reliable for favicon use. Use the gradient in `public/logo.svg` (used via `<Image>` component, rendered server-side), use solid colors in `app/icon.svg`.

Commit both files once written.
