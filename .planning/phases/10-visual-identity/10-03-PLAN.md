**Step 2: Create and run a generation script.**

Create `scripts/generate-icons.mjs` at the repo root:

```javascript
// scripts/generate-icons.mjs — run once, commit outputs, delete script afterward
import sharp from 'sharp';
import { encode } from 'sharp-ico';
import { readFileSync, writeFileSync } from 'fs';

const iconSvgPath = 'packages/web/app/icon.svg';

// Generate apple-icon.png (180x180)
await sharp(iconSvgPath)
  .resize(180, 180)
  .png()
  .toFile('packages/web/app/apple-icon.png');
console.log('apple-icon.png generated');

// Generate ICO frames
const img32 = await sharp(iconSvgPath).resize(32, 32).png().toBuffer();
const img16 = await sharp(iconSvgPath).resize(16, 16).png().toBuffer();

// Encode as ICO (multi-size container)
const icoBuffer = encode([img16, img32]);
writeFileSync('packages/web/app/favicon.ico', icoBuffer);
console.log('favicon.ico generated (16x16 + 32x32 frames)');

console.log('Done. Verify with: file packages/web/app/favicon.ico');
```

Run the script:
```bash
cd /Users/qian/Projects/Qlj/qlj-fe-i18n && node scripts/generate-icons.mjs
```

**Step 3: Verify the ICO is valid.**

```bash
xxd packages/web/app/favicon.ico | head -2
```
Expected: first 4 bytes are `00 00 01 00` (ICO magic bytes). If you see `89 50 4E 47` it's a PNG renamed to .ico — that's wrong.

If `sharp-ico` encode API differs (check its README), adapt: `encode([img16, img32])` may return a Buffer or require a filename argument.

**Step 4: Remove the script after generation** (it has no value committed).
```bash
rm scripts/generate-icons.mjs
```

If `sharp` fails to render the SVG (some versions have SVG support issues), fallback: use the online tool at realfavicongenerator.net — upload `packages/web/app/icon.svg`, download the ICO and apple-touch-icon.png, place them in `packages/web/app/`. This is an acceptable fallback for a one-time generation.
