## Design: 模块页面重设计

### 数据模型

```typescript
// Before
interface ProjectModule {
  name: string;   // 必填，标为"中文"
  code: string;
}

// After
interface ProjectModule {
  code: string;        // 主标识符，必填
  description?: string; // 可选描述，语言无关
}
```

### UI 布局

```
┌──────────────────────────────────────────────────────────┐
│  模块                                        [+ 添加]    │
│  按功能模块组织翻译 key 的命名空间                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ┌─ 添加模块（展开态）────────────────────────────────┐   │
│  │ 模块代码 [smartShield___]  描述（可选）[___]  [添加]│   │
│  │ ℹ 代码作为 key 前缀，如: smartShield.login.title   │   │
│  └────────────────────────────────────────────────────┘   │
│                                                          │
│  ┌──────────────┬──────────────┬──────┬───────┐          │
│  │ 模块代码     │ 描述          │ 词条 │ 操作  │          │
│  ├──────────────┼──────────────┼──────┼───────┤          │
│  │ smartShield  │ 智能绿盾      │  12  │  🗑   │          │
│  │ user         │ —            │   8  │  🗑   │          │
│  │ common       │ 公共翻译      │  45  │  🗑   │          │
│  └──────────────┴──────────────┴──────┴───────┘          │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 交互规则

1. **展开/折叠**：标题栏的 [+ 添加] 按钮切换表单显隐
2. **空状态**：当模块数为 0，表单默认展开，不显示列表区域的空状态占位
3. **有模块时**：表单默认折叠，列表为主角
4. **添加成功后**：表单清空但保持展开（方便连续添加）
5. **验证**：code 仍需匹配 `/^[a-z][a-z0-9_]*$/i`，description 无限制

### 迁移策略

现有 `name` 值直接迁移为 `description`，无数据丢失风险。`code` 不变，不影响任何已有的 key 前缀引用。
